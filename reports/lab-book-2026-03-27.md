## What was built

A browser-based web UI wrapping the wcEcoli whole-cell model, built in Python/Dash. Lets a user configure, run, inspect, and explore simulations entirely from a browser without touching the command line.

**Four tabs:**
- **Configure** — presets (wildtype, nutrient shifts, ppGpp sweep, acetate), variant/seed/generation controls, regulation toggles with tooltips
- **Run Status** — live job queue (SQLite-backed, background thread), phase tracking: queued → parca → simulating → analysing → done/failed
- **Inspect Data** — interactive Plotly chart; browse any listener/column; overlay multiple traces with remove buttons; normalise/log transforms
- **Explore Plots** — side-by-side gallery of the 80+ pre-generated analysis plots

## Proposed next work: platelet whole-cell model

The wcEcoli framework separates a generic simulation engine (`wholecell/`) from the E. coli model (`models/ecoli/`). The engine is reusable for other organisms. Proposal is to create `models/platelet/` as a new organism namespace.

**Phased plan (ideas — time will likely only allow some of these):**
1. Stub model — minimal molecules, one process, runnable in the existing web UI
2. Calcium dynamics — IP₃-mediated store release, SOCE, cytosolic buffering (parameterise from Purvis et al. 2008)
3. Granule release — dense/alpha granules as `UniqueMolecule` instances; exocytosis triggered by Ca²⁺ threshold
4. Receptor signalling — GPVI, PAR-1/4, P2Y₁₂ pathways converging on Ca²⁺
5. Integrin activation, metabolism (optional, lower priority)

**Key data sources needed:**
- Burkhart et al. (2012) *Blood* — quantitative platelet proteome (~4000 proteins with copy numbers) for initial conditions
- Purvis et al. (2008) *Blood* — ODE rate constants for calcium/phosphoinositide signalling; validation target for Ca²⁺ transients

**Engine changes required:** termination on time limit (not cell division), generalise compartment mass accounting for dense granules / alpha granules / mitochondria.
