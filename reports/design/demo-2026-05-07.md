   - Three platelet presets, all genuinely distinct biologically — that's

     - 🩸 IP3 transient (+Ca²⁺) — Phase 1 result (200 s, IP3 ON, 1.2 mM Ca²⁺_ex)

## What the model is (30 seconds)

- Single-cell platelet whole-cell model, 1-second discrete timesteps,
  built on the CovertLab/wcEcoli simulation framework (process / state
  / listener / ODE pattern reused; all E. coli biology removed and
  replaced).
- **27 ODE state species**: IP3R 6-state Markov (Sneyd & Dufour 2002),
  SERCA 6-state E1/E2 cycle (Dode 2002 / Purvis 2008), PMCA 5-state
  CaM-coupled (Caride 2007 Table 3), CaM Ca²⁺-binding ladder,
  STIM/Orai MWC SOCE (Hoover & Lewis 2011 / Dolan 2014), IP3 forcing
  curve (Dolan Fig. S2).
- **Validation target**: Dolan & Diamond 2014 Biophys J — the most-cited
  platelet Ca²⁺ model.

## The honest limitation (one paragraph)

The DTS empties to 0 within ~5 s under stimulation in either condition
because IP3R inflow at the published 10 pS conductance (~112 k ions/s
at full DTS) overwhelms SERCA's max forward delivery (~8 k ions/s,
hard-capped by E2P·Ca occupancy upstream of the release step). Once the
DTS is empty, IP3R has nothing left to release; the SOCE-driven plateau
that Dolan's Fig. 4 shows in the +Ca_ex condition cannot form because
the cytosolic Ca²⁺ has already collapsed by the time SOCE engages
(STIM1 dimers do rise from 22 to ~810 — the store-sensing mechanism is
working, but its product comes too late).

This is documented in **design doc §6.8 D7** and in the **2026-05-06
lab books** (path B falsification + Phase 3 results).

## Likely Q&A

**Q: Why don't the two conditions give different peaks?**
A: The peak is set by IP3R drainage of the DTS, which finishes in ~5 s
in both cases. SOCE (the only mechanism that *would* differ between
conditions) acts after that, but cyt has already collapsed.

**Q: Why not just tune the model?**
A: We tried. The single-parameter sweep on `k_release_r` (lab book
2026-05-06 morning) showed the SERCA cycle's max forward delivery is
hard-capped at 8 000 ions/s by upstream supply of E2P·Ca, regardless
of `k_release_r`. The earlier attempt to recalibrate `γ_IP3R`
(Phase 2a, 2026-05-05) ran into the symmetric problem on the other
side: any γ small enough gives DTS *overshoot* and cyt undershoot.
The system needs either a multi-parameter calibration (low-confidence)
or a missing biology compartment (better story).

**Q: What's the next step then?**
A: Mitochondrial Ca²⁺ uniporter (issue #22). Three of the 16 papers
in `source-info/calcium-papers/` are specifically on platelet MCU —
Ajanel 2025, Ghatge 2026, Shehwar 2025 — all post-Dolan and current
research direction. Mitochondria buffer Ca²⁺ during the transient and
slowly release it back over minutes; that's exactly the missing
"plateau" mechanism. Estimate 1–2 days to implement.

**Q: How does this compare to Dolan?**
A: Dolan published their model with a homeostatic-filtered population
(0.06 % of 2.6 M sampled ICs). Our v0.2 implements the same equations
with the same primary-source rate constants, but the published Dolan
IC isn't a steady state of our ODE because of cycle-throughput
arithmetic that didn't matter for Dolan's filtering work. The
deviation is fully traced and documented as a v0.2 known limitation.

**Q: Is this dissertation-grade?**
A: The peak amplitude criterion (the *primary* biological-validity
result Dolan uses) is met. The SOCE-shape criterion isn't, and that's
honest framing for a Methods/Results section.

## Commands cheat sheet

```bash
# Webapp (local)
make run                          # http://localhost:8050/
make stop

# Phase 3 reproducer (both ±Ca²⁺ conditions + comparison plot, ~60 s)
PYTHONPATH=$PWD OPENBLAS_NUM_THREADS=1 MPLBACKEND=Agg \
    pyenv exec python runscripts/manual/runPhase3.py phase3_demo --length 200

# Single sim — three biologically-distinct conditions, exposed via flags:

# (1) IP3 transient with extracellular Ca²⁺ (Phase 1, default)
PYTHONPATH=$PWD OPENBLAS_NUM_THREADS=1 \
    pyenv exec python runscripts/manual/runPlateletSim.py transient --length 200

# (2) EDTA transient (Phase 3 no-extracellular-Ca²⁺)
PYTHONPATH=$PWD OPENBLAS_NUM_THREADS=1 \
    pyenv exec python runscripts/manual/runPlateletSim.py edta --length 200 --ca-ex-mM 0

# (3) Resting (no IP3 stimulus)
PYTHONPATH=$PWD OPENBLAS_NUM_THREADS=1 \
    pyenv exec python runscripts/manual/runPlateletSim.py resting --length 300 --no-ip3-forcing

# Tests
PYTHONPATH=$PWD OPENBLAS_NUM_THREADS=1 MPLBACKEND=Agg \
    pyenv exec python -m pytest models/platelet/tests/ -q

# Type check
PYTHONPATH=$PWD pyenv exec python -m mypy models/platelet/ reconstruction/platelet/
```

### Where the conditions are defined in code

| Condition | Defined in | Read by |
|---|---|---|
| Extracellular Ca²⁺ (`--ca-ex-mM`) | `cs_mod.CA_EX_UM` (module constant) in `reconstruction/platelet/dataclasses/process/calcium_signalling.py` | `_ode_rhs` (SOCE current + PM leak, both gated on `> 0`); the `CalciumTrace` listener |
| IP3 forcing (`--no-ip3-forcing`) | `CalciumDynamics._ip3_forced` (class attribute) in `models/platelet/processes/calcium_dynamics.py` | passed into `cs.molecules_to_next_time_step(... ip3_forced=...)` each timestep; the ODE applies `ip3_forcing_uM(t)` (Dolan Fig. S2 fit) when True |
| Webapp presets | `wholecell/webapp/tabs/configure.py:PRESETS` | the Configure-tab callback fills the form fields, jobs.py threads them into the runscript flags |
